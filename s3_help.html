Getting S3-Compatible Cloud Storage Credentials

Unlock requires S3-compatible cloud storage to save generated packet content, like summaries and quizzes. This allows your packets to be self-contained and shareable. Unlock supports AWS S3, DigitalOcean Spaces, and Google Cloud Storage.

General Steps:

  1. Choose a Provider: Decide which provider you want to use.
  2. Create a Bucket/Space:
    • In your provider's console, create a new **Bucket** (AWS/Google) or **Space** (DigitalOcean).
    • Choose a globally unique name (e.g., `unlock-data-yourname`).
    • Select a **Region** (e.g., `us-east-1`, `nyc3`). Remember this name.
    • **Crucially, disable "Block all public access"** (for AWS/Google) or ensure files can be made public. Unlock needs this permission to create shareable links for your generated content.
  3. Generate Access Keys:
    • Find the section for managing access keys. For S3 compatibility, these are often called **HMAC keys** in Google Cloud.
    • Generate a new set of keys. This will give you an **Access Key ID** and a **Secret Access Key**.
    • Copy both keys immediately and store them securely, as the secret key is often shown only once.
  4. Enter Credentials in Unlock:
    • Go to the Unlock sidebar Settings.
    • Add or edit a Storage Configuration.
    • Enter the **Access Key ID**, **Secret AccessKey**, **Bucket/Space Name**, and **Region**.

Provider-Specific Notes

AWS S3

  • Go to the AWS S3 Console.
  • Create your bucket and uncheck "Block all public access" under the bucket's permissions.
  • Generate keys in the **IAM** (Identity and Access Management) service. Create a new user with programmatic access and attach the `AmazonS3FullAccess` policy, or a more restrictive custom policy that allows `s3:GetObject`, `s3:PutObject`, `s3:DeleteObject`, and `s3:ListBucket` for your specific bucket.

DigitalOcean Spaces

  • Go to the Spaces section in your DigitalOcean account.
  • Create a Space. File listing should be public if you intend to share widely, but files themselves are private by default.
  • Generate keys under the **API** section in your account, under "Spaces access keys".
  • The region will be like `nyc3`, `sfo3`, etc.

Google Cloud Storage

  • Go to the Cloud Storage section in your Google Cloud project.
  • Create a bucket. During creation, under "Choose how to control access to objects", select **"Fine-grained"**.
  • After creation, go to the bucket's "Settings" tab, then the "Interoperability" sub-tab.
  • Under "Access keys for service accounts", create a key for a user or service account. This will generate your S3-compatible **HMAC keys** (Access Key and Secret).
  • The region for Google Cloud Storage is typically `auto` or a specific location like `US-EAST1`. For global access, you can often use `global`.

Note: Cloud provider interfaces change. Always consult the official documentation for your chosen provider for the most up-to-date instructions.
